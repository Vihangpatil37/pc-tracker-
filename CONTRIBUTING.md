# Contributing to FocusOS

First off, thank you for considering contributing to FocusOS! People like you make open-source a great place.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please use our Bug Report template in the issue tracker. Include as much detail as possible, including operating system version and reproduction steps.

### Suggesting Enhancements
Feature requests are always welcome. Please use the Feature Request issue template or open a discussion in GitHub Discussions.

### Code Contributions
1. Fork the repository.
2. Clone your fork locally.
3. Check out our `docs/getting-started.md` for instructions on setting up your local environment (Node.js + Rust).
4. Create a new branch for your feature or bugfix (`git checkout -b feature/amazing-feature`).
5. Commit your changes (`git commit -m 'Add some amazing feature'`).
6. Push to the branch (`git push origin feature/amazing-feature`).
7. Open a Pull Request!

## Development Guidelines
- Make sure all Rust and Vitest tests pass locally before submitting a PR.
- Please follow our architecture guidelines found in the `docs/` folder.
- Ensure your code is formatted properly using `cargo fmt` for Rust and `npm run format` (or equivalent) for TypeScript.

We look forward to reviewing your contributions!
